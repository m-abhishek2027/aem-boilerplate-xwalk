Phase 1 — GitHub Repository

Step 1: Create empty repo on GitHub

Name: aem-boilerplate-xwalk (Public, no README)

Important: Use aem-boilerplate-xwalk, NOT plain aem-boilerplate for Universal Editor.

Step 2: Import Adobe template code

cd D:\eds
git clone https://github.com/m-abhishek2027/aem-boilerplate-xwalk.git
cd aem-boilerplate-xwalk
git remote add template https://github.com/adobe-rnd/aem-boilerplate-xwalk.git
git fetch template main
git checkout -f -B main template/main
git push -u origin main

Step 3: Local install

npm install
npm run lint

Phase 2 — Connect AEM Sandbox (fstab.yaml)

Edit fstab.yaml in repo root:

mountpoints:
  /:
    url: "https://author-p139816-e1765605.adobeaemcloud.com/bin/franklin.delivery/m-abhishek2027/aem-boilerplate-xwalk/main"
    type: "markup"
    suffix: ".html"

Replace author-pXXXX-eYYYY with your AEM Author host.

git add fstab.yaml
git commit -m "Connect AEM sandbox author as content source"
git push origin main

Phase 3 — AEM Code Sync

  1. Open: github.com/apps/aem-code-sync
  2. Select Only select repositories
  3. Choose aem-boilerplate-xwalk
  4. Save → confirm "AEM Code Sync registered"

Phase 4 — User Admin

  1. Open: tools.aem.live/tools/user-admin
  2. Sign in to project → allow pop-ups
  3. Org: m-abhishek2027, Site: aem-boilerplate-xwalk
  4. Fetch Users → Add your AEM email as Admin

Phase 5 — paths.json (Universal Editor Fix)

Required: Without this file, Universal Editor shows paths.json 404 Not Found error.

Create paths.json in repo root:

{
  "mappings": [
    "/content/aem-boilerplate-xwalk/:/"
  ],
  "includes": [
    "/content/aem-boilerplate-xwalk/",
    "/content/dam/aem-boilerplate-xwalk/"
  ]
}
git add paths.json
git commit -m "Add paths.json for Universal Editor"
git push origin main

Verify: main--aem-boilerplate-xwalk--m-abhishek2027.aem.page/paths.json shows JSON (not 404).

Phase 6 — Create AEM Site

  1. Download template ZIP from github.com/adobe-rnd/aem-boilerplate-xwalk/releases
  2. AEM Author → Sites → Create → Site from template → Import ZIP
  3. Create site with these values:
FieldValue
Site nameaem-boilerplate-xwalk
Project URLhttps://main--aem-boilerplate-xwalk--m-abhishek2027.aem.page

Phase 7 — Universal Editor

  1. Sites → aem-boilerplate-xwalk → index → Edit
  2. Sign in with Adobe
  3. Add/edit components (Cards, Hero, etc.)

Phase 8 — Publish

  1. AEM → Tools → Cloud Services → Edge Delivery Services Configuration
  2. Properties → Authentication → copy Technical Account ID
  3. User Admin → Add technical account as Config Admin
  4. Universal Editor → Publish → Destination: Preview
  5. Check preview URL loads correctly

Phase 9 — Local Development

cd D:\eds\aem-boilerplate-xwalk
npx @adobe/aem-cli up

Open: http://localhost:3000/

Optional: Local draft content

npx @adobe/aem-cli up --html-folder drafts

Create drafts/index.plain.html for local testing without AEM content.

Phase 10 — Sidekick (Optional)

Install from Chrome Web Store: AEM Sidekick → Pin → Add project on preview URL.
